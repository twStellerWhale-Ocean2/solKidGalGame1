# I. 主旨目的

定義「自架網頁遊戲（含線上管理）」這一**系統類型**：家庭/小群體自架伺服器、玩家帳號雲端存檔、
維護者線上管理的網頁遊戲。一經宣告即綁定：(1) 最低能力清單、(2) **帳號/角色動線 canon**、
(3) 雙端介面水準 bar、(4) **標準 mod 組成**與**單一 release 部署組成**、(5) 點名 cmpTechItem。
sys 顆粒判準依 GLOSSARY（≈微服務：本型 sys 擁有玩家資料所有權與 `/api/*` 對外契約、獨立演進）。

# II. 參考準備

* **適用場景**：一家一伺服器的家庭/小群體網頁遊戲，兒童向與一般向皆適用——
  兒童向：玩家=兒童（家長協助）、維護者=家長；一般向：玩家=一般使用者、維護者=站主。
  目標客群以 design.md ＜I＞ 宣告為準，並決定下列條件項是否觸發。
* **公開參照**：不重造輪子——帳號體系（bcrypt、opaque session）、樂觀併發（updatedAt）、
  護眼/家長管控慣例、MD3 管理網站慣例（管理端）。
* **四層鏈**：sol 之 techStyle 定風格 → 本契約定 sys 組成 → 各 mod 綁 techStack → cmpTechItem 進 mod。

# III. 內容程序

## A. 最低能力清單（GATE ＜5節＞ 鏡頭 A 逐項對照）

* **玩家帳號與雲端存檔**：註冊/登入/登出、session 快取續玩、跨裝置還原、樂觀併發不靜默覆蓋、
  離線韌性（伺服器暫不可達可續玩、背景重試、明確同步提示）。
* **帳號內多角色**：一帳號可建多角色（profile／character slot）、各自進度、可切換；
  動線依 ＜B＞ canon；角色上限、「守最後一員」等定值由該 repo design.md 定。
* **玩家端可用性**：手機直向第一視口、觸控 ≥44px；**兒童向另須**錯誤訊息兒童友善、家長協助輸入動線。
* **護眼/家長管控（條件項：僅兒童向適用，一般向 N/A）**：遊玩時長限制與強制休息；
  維護者可鎖定個別帳號時長（管控屬家長輔助，非防駭防線）。是否觸發依 ＜II＞ 宣告之目標客群。
* **維護者線上管理**（管理網站常規，勿只顧遊戲域功能）：登入/登出、帳號清單、線上重設密碼、
  撤銷登入、刪除帳號（二次確認、防自鎖）、執行期設定（儲存即生效）、關於/版本、使用說明入口。
* **安全基線**：密碼單向雜湊、統一錯誤不洩帳號存在性、速率限制僅計失敗、受保護 API 一律驗 session、
  管理 API 一律驗 admin 身分、靜態服務 allowlist（不外洩原始碼樹）。
* **維運**：健康檢查（/healthz、/readyz）、資料庫備份還原程序記入產品手冊。

## B. 帳號/角色動線 canon（Web 遊戲登入標準模式；兒童向與一般向通用）

* **兩層模型（硬性）**：帳號（Account）＝管理/持有層——密碼把關、一台裝置登入一次即**保持登入**
  （不自動登出、不逐次要密碼）；角色（Character/Profile）＝遊玩層——一帳號多角色、各自進度。
  「你是誰」只在角色層回答。對標：串流平台 profile 模式（Netflix/Disney+）、主機開機選人；
  兒童向＝家長帳號＋小孩角色，一般向（青少年 RPG 等）＝玩家帳號＋多角色欄位。
* **表單恆兩張**：
  1. **登入表單**（僅未登入時出現）：帳號＋密碼＋註冊入口；登入成功即記住本裝置。
  2. **選角色頁**（登入後家門口，每次上線經過）：頂＝帳號名稱/資訊＋登出；
     中＝角色清單（大頭照＋名字，逐角色：檢視資訊／進入遊戲／刪除角色）；底＝新增角色。
* **角色密碼（選配）**：新增角色時可選設（防共用裝置互玩，可留空）；有設→進入遊戲/刪除角色須驗證；
  未設→點即進入、刪除走兩段防呆。刪除帳號屬管理端職權，不入玩家端。
* **切換角色**＝自遊戲返回選角色頁（單一路徑，不另設第二套切換 UI）。
* **反樣式（出現即回修）**：登入頁並列多張帳號卡各自要密碼（辦公多帳號共機模式，與遊戲場景錯位）；
  「移除本機帳號卡」類裝置管理鈕曝露於遊玩主動線；「你是誰」在帳號層與角色層各答一次（雙層互打）。
  ——起因：solLingoWorld ≤v0.74.1 增量疊加成帳號卡＋角色 roster 雙層錯位，USR 裁定收斂為本模式。

## C. 介面水準 bar（GATE ＜5節＞ 鏡頭 C 逐頁判讀；雙端分流）

* **玩家遊戲端**：沿 sol 之 techStyle 全幅藝術視覺（不套管理網站皮）；不得出現任何管理/dev 入口。
* **管理端**：MD3 管理網站基座＋techStyle token；行動視口可用（表格降級卡片列）；
  危險操作 error 色＋二次確認、回饋 snackbar、未儲存變更防護。
* **不合格樣式（出現即回修）**：兒童端曝露管理入口、管理端硬寫色/原生 alert、
  截圖證據混入 dev-only 元素、鎖定/唯讀狀態無視覺標示。

## D. 標準 mod 組成（每 mod 綁一個 techStack，圖以文字標記 `techStack: XXX`）

| mod | techStack | 職責 |
|---|---|---|
| modShell | StaticWeb | 遊戲殼（無框架 ES modules；內容包同屬此構件交付） |
| modAdmin | StaticWeb（複雜化時 ReactWeb，走 ADR） | 線上管理頁（/admin/ 靜態子樹） |
| modApi | NodeSvr | 帳號/存檔/管理 API＋靜態服務 allowlist |
| modDb | Postgres（現成服務型） | 玩家資料與執行期設定；所有權屬本 sys |

* 點名 cmpTechItem：無硬性強制（語音類走瀏覽器內建 Web Speech；如引入外部 TTS/評分再點名對應 cmpTechItem）。

## E. 部署組成（單一 release）

* 部署選型 `deployHelm`：**一個 sys＝一個 helm chart**——modApi image（烘入 modShell/modAdmin 靜態包，
  同站服務免 CORS）＋ modDb（chart 依賴或實體寄宿平台叢集，寄宿不改所有權）。
* 過渡期（helm 化前）等效形態：compose 資料庫＋node 服務同站，允許為增量中繼態。
* TLS：藏 gateway 後終結；內網 HTTP 過渡須明文風險與「勿轉發公網」警語。
