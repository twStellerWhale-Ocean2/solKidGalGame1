# 通用性規範

# I. 主旨目的

本規範整理所有 HMI / UI / UX 介面共同遵守的視覺、互動、可用性與 AI 生成**原則**，適用於系統、模組、子平台、Dashboard、MIS、AI 平台、管理介面、說明網站、Figma 畫面與前端工程元件。

本規範以 Google Material Design 3 為主要外部參考框架，沉澱 AI 科技新創風格、Enterprise MIS、Desktop-first Dashboard 與高資訊效率設計原則。

**本規範定原則與不變量，不手寫死數值。** 間距、字級、radius、色票等具體數字由**主題 token 定本 + MUI theme / MD3** 承載（生成一次、commit 為定本、唯讀引用，見 [techStackReactWeb]／[techStackStaticWeb]），本規範不複述以免與 token 漂移、或綁死框架發揮。具體版面細則依情境見專用性規範（管理網站／說明網站／命令列）。

# II. 參考準備

## A. Google Material Design 3

MD3 為 Base Framework，用於一致的元件語言、互動狀態、版面節奏、色彩階層與可維護設計系統；借用其 component／token／motion／state／accessibility／layout 概念，轉化為企業 MIS／AI Dashboard 可長期維護的介面，而非消費型 App。

參照時優先確認：Material component 的狀態與場景；Top App Bar／Navigation Drawer／Dialog／Button／Text Field／Table 的互動慣例；Color token／Surface／Container／State layer 的層級；8dp 節奏與一致 radius／elevation；Dark theme 與高密度介面可讀性。

## B. Material Symbols Rounded

Icon 統一採 Material Symbols Rounded，避免混用多套造成語意、線條粗細、圓角與視覺重量不一致；同一工具列／元件區域內尺寸一致。

若 Rounded 無法涵蓋特殊情境，先找同風格、同語意密度的 icon 再評估納入；不得為單一功能臨時混入不同視覺語言的 icon。選用外部 icon 須檢查授權、線條粗細、圓角端點、填色方式，並確保同頁只用一套主要 icon 語言。

## C. Design Token 與 8dp 節奏

版面以 **8dp grid 節奏**維持間距一致，避免隨意 spacing 造成低品質感。色彩、字型、間距、radius、surface 等以 **Design Token** 結構化，供 Figma／CSS／元件庫／AI 生成共用；token 由種子生成、commit 為定本後唯讀引用（不在本規範列舉數值）。

## D. WCAG 與無障礙

至少參照 WCAG 2.1 AA。檢查：Keyboard Navigation、Focus State、Contrast Ratio、Readable Font、表單錯誤訊息可辨識、互動元件有足夠點擊區域與狀態回饋。

# III. 內容程序

## A. 整體設計哲學

呈現 AI 科技新創與 Enterprise Dashboard 混合風格：專業、科技感、冷靜、高效率；核心為可掃描、可快速操作、低認知負荷、高資訊效率、高可維護。

採用風格：Material Design 3、AI Startup、Enterprise Dashboard、Dark Operation Center、Cyber／Data-driven。
禁止風格：過度卡通化、過度玻璃擬態、彩虹色、過度動畫、過度陰影、過度漸層、過度裝飾性 UI。

> 上述偏 Dashboard／Dark 取向為**預設基調**；說明網站等情境由其專用性規範覆寫（可較敘事），不得反向把管理介面行銷化。

## B. 色彩系統與主題

**支援 system／light／dark 三態切換，預設跟隨系統**；各主題色由 token 定本提供，不在此寫死色票。

主色落在 **Cyan Blue／AI Blue／Teal／Deep Blue** 等科技感但不刺眼的色域（品牌種子色約束，具體值見 token 定本）；語意色維持 Error／Success／Warning／Info 一致。**禁止**高彩度彩虹配色、高亮紅、過度鮮豔或螢光背景。

## C. Typography

字體統一 Inter／Roboto／Noto Sans TC，**禁止**藝術字、手寫字與 Serif。建立清楚字級階層（具體字級由 token／theme scale 提供），**一頁最多三種字級層次**；標題不過大、不浮誇字距，內容優先可讀性。

## D. Spacing、Radius 與邊界

遵循 **8dp grid 節奏**，**禁止**無規則 spacing（如 13、17、21、27）。Radius／elevation 全站一致（依 token，不逐元件列 dp）。Border 以細線為主，**禁止**粗黑框、多重邊框與重陰影。（卡片 padding、section gap、各元件 radius 之實際值由 token／theme 提供。）

## E. 動畫與互動回饋

動畫快速、輕量、不干擾操作（duration 由 token／theme 提供，不在此寫死毫秒）。允許 Hover、Ripple、Dialog Fade、Drawer Slide；**禁止**過度縮放、旋轉與過長 transition。

## F. 響應式規範

優先順序 **Desktop First → Tablet → Mobile**。Desktop 為主要操作平台；Mobile 允許 Collapse Drawer 與 Bottom Navigation，但**禁止**將 Desktop layout 強行縮小。

手機／rendered UI 驗收以**實機有效可視區**為準，不得以理想 viewport 宣稱通過：

- 須算進實機 browser chrome、地址列、系統列造成的有效高度差；不得把 `390x844` 之類理想 viewport 等同實機可視高度。
- 關鍵元件（返回鍵、主要 action、底部導覽）須以 `window.innerHeight`／`window.visualViewport.height` 與 bounding box 驗證落在有效可視範圍；只確認 DOM 存在不算通過。
- 驗收 URL 應與使用者實際所見一致（production／Pages vs localhost），並確認 commit／build／cache 相同。
- 實機截圖與開發端不一致時以實機為準，先處理驗證條件差異，不得以本地 headless 截圖反駁實機。

## G. AI 科技感控制

允許微量 Glow、Cyan Accent、Grid Background、Data Visualization、Thin Line；**禁止** Cyberpunk 過度風格、過度霓虹與過度特效。科技感服務於資訊辨識與操作效率，不壓過資料本身。

## H. AI UI 生成規範

AI 生成 UI 必須：用 MD3、依本規範 token 與 8dp 節奏、用 Material Symbols Rounded、一致 radius／typography、乾淨專業企業級、高資訊密度但可讀。
禁止：以 Fancy Landing Page 取代產品介面、Marketing Style 取代管理系統、多彩漸層、浮誇動畫、Dribbble Style、過度玻璃擬態。
