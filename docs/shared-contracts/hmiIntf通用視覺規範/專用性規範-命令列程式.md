# 專用性規範-命令列程式

# I. 主旨目的

本規範適用於 PowerShell stage script、部署腳本、Secret 匯入腳本、測試腳本、release 腳本與其他給開發者或維運者執行的 CLI。

CLI 是產品操作介面的一部分，不得只以「能跑」為完成標準；輸出必須能讓操作者理解目前階段、輸入來源、風險、錯誤、中止原因與完成結果。

# II. 參考準備

* [ ] 參考 `9general-create-pwshscript/SKILL.md` 的 PowerShell 腳本編寫規則。
* [ ] 參考 `skill-2tech-2develop-stagescript-3packpkg-importk8ssecret/importK8sSecret.ps1` 的章節輸出、參數提示、ENV/User/K8S 讀寫與確認流程。
* [ ] 若 CLI 會讀寫 Secret、ENV、K8S、GHCR、Docker、Helm 或外部服務，先確認哪些值可顯示、哪些必須遮罩。

# III. 內容程序

## A. CLI 章節結構

PowerShell CLI 應以清楚章節呈現：

```text
# I. 主旨目的
# II. 參考準備
## A. 固定參數與資料準備
## B. 目前狀態檢查
# III. 內容程序
## A. 執行主要作業
## B. 驗證結果
# IV. 備註紀錄
```

階段輸出應使用穩定標題與顏色，讓操作者能掃描目前進度。可參考：

```powershell
Write-Host '# I. 主旨目的================================' -ForegroundColor Blue
Write-Host '## A. 固定參數與資料準備--------------------------------' -ForegroundColor Cyan
```

## B. 顏色語意

建議配色：

```text
Blue: 主章節
Cyan: 子章節
Green: 成功、已完成、已存在且可用
Yellow: 警告、採用預設、可繼續但需注意
Red: 錯誤、中止、缺少必要條件
Gray/DarkGray: 補充資訊、遮罩後資訊、非關鍵細節
```

顏色不得作為唯一資訊來源；重要狀態仍需有文字。

## C. 參數與輸入

CLI 應明確列出必要參數、預設值來源與實際採用值。

若腳本支援互動與非互動模式，應提供明確參數，例如：

```powershell
param(
  [switch]$NonInteractive
)
```

互動模式可讓使用者確認或覆寫；非互動模式缺少必要值時必須告警中止，不得使用假值或無來源預設值。

## D. ENV、User 層與 Process 層

涉及環境變數時，應同時考慮 process 層與 user 層。讀取順序需明確，常見做法為：

```text
目前 process ENV
User 層 ENV
K8S Secret 現值
互動輸入或手動指定值
```

若將值寫入 User 層 ENV、目前 pwsh session 或 K8S Secret，必須在寫入後重新讀取並顯示更新結果。

## E. Secret 與敏感值

真實 password、token、API key、client secret、private key 不得明文輸出。

輸出時應使用遮罩：

```text
OPENAI_API_KEY: sk-...abcd
CLIENT_SECRET: ********
```

測試帳號、測試角色、測試客戶、測試訂單與 README 明確定義的測試資料不視為敏感值；但若包含真實憑證或可連到正式外部服務，仍必須遮罩。

## F. 錯誤與中止

腳本應使用可預期的錯誤策略：

```powershell
$ErrorActionPreference = 'Stop'
```

缺少必要條件時必須列出：

- 缺少哪個值。
- 目前查過哪些來源。
- 這會影響哪個 stage 或測試。
- 使用者可如何補齊。

不得吞掉錯誤後顯示成功。

## G. 完成摘要

CLI 結束時須提供摘要：

- 已完成哪些作業。
- 寫入或更新哪些資源。
- 驗證哪些結果。
- 哪些項目被跳過或未完成。
- 下一步建議。

若有 `SkipDeploy`、`SkipOciPush`、`NonInteractive`、降級或未完整 release，摘要中必須明確標示，不得讓操作者誤以為完整完成。

## H. 視覺一致性

同一 REPO 的 stage script 應使用一致的章節名、顏色語意、錯誤格式、摘要格式與參數命名。

若腳本會開啟瀏覽器或展示頁面，CLI 應明確告知 URL、port-forward、namespace、release、測試帳號與展示模式。
