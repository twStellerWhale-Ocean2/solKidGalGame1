# design-issue181 — 各地區打工報酬難度級距再收斂設計note

> 本檔為 issue #181 的 plan 設計note（非 design.md 正本、不受 docLint），是「打工報酬難度級距再收斂」的**單一落地依據**。
> design.md 對應：`spec#11`（打工報酬採平緩等差級距、隨地區英文難度微幅遞增）已涵蓋本案語意，**design.md 文字不變**；本案僅再收斂報酬數值，數值落點集中於本檔。
> **取代關係**：本檔 §2 報酬表**取代** [docs/design-issue157.md] §2（城堡 50／市區 60／鄉村 70／森林 80）為現行報酬依據；design-issue157.md §3 商店定價**仍為現行、本案不動**。

---

## 1. 修訂定位與緣起

前案 #157 才於 2026-06-18 把四地區打工報酬由陡峭階梯 `城堡 20 / 市區 100 / 鄉村 500 / 森林 2000` 壓為公差 10 之平緩等差 `城堡 50 / 市區 60 / 鄉村 70 / 森林 80`（最大 1.6×）。USR 認為 1.6× 之地區差距仍可能誘導兒童玩家「只為了賺錢就每次都跑去最難的地區硬猜」，要求差距更小——「有感但不大」（緣起見 Issue #181＜I＞）。

本次把報酬**再收斂**為基數 `100`、公差 `5` 之平緩等差 `城堡 100 / 市區 105 / 鄉村 110 / 森林 115`（最大僅 1.15×）：地區報酬差距幾近齊平、移除「衝高難度地區賺錢」誘因，同時整體墊高報酬絕對量級。

**保留（不動）：**

1. 四大地區與英文分級（Castle Dolch、Urban Starters、Rural Movers、Wild Flyers）及其由易至難之順序。
2. 場景、角色、互動模式與資料結構欄位（`reward`、`cost` 等鍵名與型別不變）。
3. 助攻獎勵階梯（中文協助 → 全額／半額／無，issue #73，`paramRewardSecondTryRatio=0.5`）——與地區報酬基數正交，不動。
4. 生活聊天 `chatReward = { coins: 0 }`（聊天不發 coins、改加心情）——不動。
5. **商店定價**：沿用 #157 已對齊之地區級距帶（design-issue157.md §3），本案**不重算**。

## 2. 打工報酬（每題答對 coins）

地區報酬改為**基數 100、公差 5 之平緩等差**（最大 1.15×，難度序沿用既有英文分級由易至難）：

| 地區 | id | 難度序 | #157 報酬 | **本案新報酬** |
|---|---|---|---|---|
| 城堡 | castle | 第一難度（Dolch） | 50 | **100** |
| 市區 | urban | 第二難度（Starters） | 60 | **105** |
| 鄉村 | rural | 第三難度（Movers） | 70 | **110** |
| 森林 | wild | 第四難度（Flyers） | 80 | **115** |

**落地欄位**（每地區兩處，須同步）：各 area manifest 之
`{area}VocabularyProfile.rewardCoins` 與 `const jobReward = { coins: N }`。
（`rewardCoins` 目前未被引擎結算直接讀取，仍維持與 `jobReward` 同值以免漂移；單一來源整併屬技術債、不在本案範圍，續列待辦。）

## 3. 商店定價（本案不動，沿用 #157）

維持 [docs/design-issue157.md] §3 之定價帶（單品約 80–290、套裝約 260–680，全部數題勞動可負擔），**本案不重算**。理由：

1. 報酬量級上提後，各地區商品在「勞動題數」上**更易負擔**（方向是更便宜、非更貴），仍落在「數題勞動內可負擔」原則內。
2. 報酬已近齊平（1.15×），地區定價階梯（#157 對齊之 1.0/1.2/1.4/1.6）不再造成「衝高報酬地區」誘因，無重算必要。
3. #181 緣起為「縮小報酬地區差距」；定價屬 Issue #181 備註所列「後續再處理」、不在本案範圍。

影響評估留 QA：實機確認各地區典型商品仍於數題勞動內可負擔、無因報酬墊高而導致顯著失衡。

## 4. 不在本案範圍（連動待辦，列明不做）

1. **商店定價重算**：不動，沿用 #157（design-issue157.md §3）。
2. **`jobReward`／`rewardCoins` 單一來源整併**：既有重複欄位技術債，本案僅兩處同步調值、不重構，續列待辦。
3. **集中「地區報酬級距表」常數化**：可降日後再平衡多檔同步成本，本案不導入、列待辦。
4. **助攻獎勵階梯（full／half／none）、聊天回饋、護眼計時**：與報酬基數正交，不動。
5. **既有存檔玩家既有 coins 餘額**：不重置；新報酬自本次起對所有帳號生效。

## 5. QA 清單（code 階段驗收）

1. 四地區 `jobReward.coins` 與 `{area}VocabularyProfile.rewardCoins` 皆為 `100/105/110/115` 且兩處同值。
2. 報酬呈公差 5、單調遞增、最大 1.15×（城堡 100 ≤ 市區 105 ≤ 鄉村 110 ≤ 森林 115），無單一地區畸高。
3. 聊天題庫 `reward.coins` 仍為 0（selftest chat／data-audit 不破）。
4. 商店各商品 `cost` 全數未變（沿用 #157）。
5. 實機抽查：以新報酬於四地區各答數題，確認典型商品於數題勞動內可購；地區典型實得 coins 呈 城堡 ≤ 市區 ≤ 鄉村 ≤ 森林 之近齊平遞增、無單一地區畸高。
6. repoLint 0／docLint(sol) 0／manifest `node --check` 通過。
